Nested Scroll Layout

This page uses a dedicated scrollable content panel instead of normal window scrolling.

Use it to inspect whether the translated side remains reasonably synchronized in split view.

Testing Goal

Many modern web applications render long documents inside an internal container. This breaks simplistic scroll listeners that only watch window. The fixture exists to expose that failure mode quickly.

Manual Steps

  1. Start translation in split view.
  2. Scroll this panel slowly to the middle.
  3. Jump near the bottom with the scrollbar.
  4. Switch to single translated view and back.

Long Content Block A

Stable regression pages should be visually obvious. If the synchronized pane lags far behind, jumps unpredictably, or stops moving entirely, the problem should be visible without opening developer tools.

Long Content Block B

Translating long content often changes total block height. A percentage-based mapping is usually more resilient than pixel-perfect mirroring, but it may still drift when the translated document becomes much taller than the original.

Long Content Block C

Use this section to see whether headings, ordered lists, and prose paragraphs remain aligned well enough to keep reading comfortable in split view mode.

Long Content Block D

The exact implementation does not need to be perfect. It only needs to be stable, explainable, and easy to regress-test after future changes.
