# snapshots 目录说明

这个目录用于存放浏览器“另存为 -> 网页，全部”得到的原始页面快照。

## 支持格式

支持浏览器默认保存出来的结构，不要求你二次清洗：

- `some-page.html`
- `some-page_files/`

只要主 HTML 文件和同名资源目录放在同一层级，并保持相对路径不变，就可以直接用于本地离线测试。

## 当前建议

- 优先把真实线上页面原样保存到这里。
- 手工构造、最小可复现的测试页继续放在 `fixtures/` 根目录。
- `snapshots/` 已加入 `.gitignore`，默认不提交这些大体积快照文件。

## 使用方式

在项目根目录启动本地服务：

```bash
python3 -m http.server 8123
```

然后直接打开对应快照页面，例如：

```text
http://127.0.0.1:8123/fixtures/snapshots/some-page.html
```

或者打开：

```text
http://127.0.0.1:8123/fixtures/index.html
```

测试首页会自动扫描 `snapshots/` 目录，并列出新增的快照页面入口。

## 备注

- 有些页面保存后仍可能依赖少量线上资源或脚本。
- 对本项目来说，这类页面的主要用途是做翻译、布局、滚动同步等回归验证，而不是要求离线页面与线上站点逐像素一致。
