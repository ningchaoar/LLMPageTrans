# 离线测试页说明

这组页面用于本地手工回归，目标是让页面内容固定、结构稳定，避免直接依赖线上站点。

## 启动方式

在项目根目录执行：

```bash
python3 -m http.server 8123
```

然后在浏览器打开：

- `http://127.0.0.1:8123/fixtures/index.html`

## 建议回归顺序

1. 打开 `index.html`，逐个进入测试页。
2. 使用插件翻译页面。
3. 观察是否成功出现双栏译文或单栏译文。
4. 开启“测试调试信息”时，记录文本节点数、批次数、耗时。
5. 对照 `regression-checklist.md` 做手工验收。

## 当前测试页

- `article-basic.html`
  - 覆盖：普通长文章、引用块、列表、侧边提示卡片
- `tech-doc.html`
  - 覆盖：标题层级、代码块、表格、公式行内文本
- `internal-scroll.html`
  - 覆盖：内部滚动容器、长列表、分屏滚动同步
- `dynamic-content.html`
  - 覆盖：延迟加载内容、展开区域、状态卡片

## 浏览器原始快照

除了手工编写的 fixture 之外，项目也支持浏览器“另存为 -> 网页，全部”的原始保存结果。

支持的目录结构如下：

- `fixtures/snapshots/some-page.html`
- `fixtures/snapshots/some-page_files/`

说明：

- 不要求二次清洗，可以直接作为离线回归样本使用。
- 需要保持主 HTML 文件与同名资源目录的相对位置不变。
- 统一使用 `.html` 后缀。
- `snapshots/` 默认加入 `.gitignore`，用于存放本地测试素材，不提交到仓库。
- 如果某些资源仍依赖线上接口，页面可能不是 100% 完整复现，但仍然适合作为翻译回归样本。
- `fixtures/index.html` 会自动扫描 `snapshots/` 目录并展示这些页面入口，不需要手动输入 URL。

## 说明

- 这些页面默认用英文内容，便于测试翻译到中文的主路径。
- 样例会持续增加，后续可以把你人工提供的页面也整理到这里。
- 如果需要更稳定的结果，测试时建议关闭浏览器翻译、阅读模式和其他修改页面结构的扩展。
