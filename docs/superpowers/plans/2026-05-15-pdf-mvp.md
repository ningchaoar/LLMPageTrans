# PDF MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a text-layer PDF translation MVP in a separate extension page that reuses the existing provider, estimate, translation, and brutal-test logging pipeline.

**Architecture:** Add a PDF-specific extension page backed by local `pdf.js`. Keep parsing helpers pure and tested in Node, while browser-only loading and translation orchestration live in `pdf-viewer.js`. Wire popup PDF detection to launch the viewer without changing the existing webpage translation flow.

**Tech Stack:** Chrome MV3 extension pages, local `pdf.js`, plain JavaScript, Node syntax checks, a tiny Node test runner.

---

## File Structure

- Create `pdf-text-utils.js`: pure grouping/map helpers shared by browser and tests.
- Create `pdf-viewer.html`: extension page markup.
- Create `pdf-viewer.css`: reader layout.
- Create `pdf-viewer.js`: PDF loading, extraction, estimate, translation UI.
- Create `tests/pdf-text-utils.test.js`: behavior tests for grouping and text maps.
- Create `tests/run-tests.js`: no-dependency Node test runner.
- Create `vendor/pdfjs/pdf.mjs` and `vendor/pdfjs/pdf.worker.mjs`: local PDF.js assets.
- Modify `popup.js`: detect PDF tabs and open `pdf-viewer.html`.
- Modify `README.md`: document PDF MVP and limitations.

## Task 3: Build PDF Viewer Page

**Files:**
- Create: `pdf-viewer.html`
- Create: `pdf-viewer.css`
- Create: `pdf-viewer.js`

- [ ] **Step 1: Add page shell**

Create toolbar, status, estimate panel, progress text, and page list containers.

- [ ] **Step 2: Implement PDF load and extraction**

Load `source` from query params, configure `pdfjsLib.GlobalWorkerOptions.workerSrc`, call `getDocument`, extract each page text, group items with `groupPdfTextItems`, and render original blocks.

- [ ] **Step 3: Implement estimation**

Build full-document `textMap`, send `estimate_translation_cost`, and render key estimate fields.

- [ ] **Step 4: Implement page-by-page translation**

Send each page's `textMap` to `translate_text`, render returned translations beside original blocks, and finalize brutal-test logs at the end.

- [ ] **Step 5: Add textless PDF messaging**

If total extracted characters is below a small threshold, show a scanned-PDF limitation message and disable translation.
